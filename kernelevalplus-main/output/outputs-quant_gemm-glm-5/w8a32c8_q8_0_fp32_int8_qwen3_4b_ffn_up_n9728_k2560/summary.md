# W8A32C8 Q8_0 FP32_INT8 GEMM Kernel - Qwen3-4B FFN Up/Gate

## 1. 任务概述

### 算子规格
- **算子类型**: quant_gemm
- **变体**: W8A32C8
- **模型**: Qwen3-4B
- **层类型**: FFN Up/Gate projection
- **维度**: N=9728, K=2560, block_size=32

### 量化格式
- **权重**: Q8_0 (block_q8_0)
  - 34 bytes/block: 2 bytes FP16 scale + 32 INT8 values
- **激活**: FP32，运行时动态量化为 Q8_1 风格
- **计算公式**: `result = d_w × d_a × sumi`

## 2. 优化历程

### v1: 基础实现
- 三种内核分别处理 M=1、小批量(M≤16)、大批量
- M=1: Warp 级并行，激活值缓存到共享内存
- 小批量: 每个 warp 计算一个输出元素
- 大批量: 权重分块缓存到共享内存

**性能结果**:
| M | 延迟 (ms) | TFLOPS |
|---|-----------|--------|
| 1 | 0.023 | 2.122 |
| 8 | 0.225 | 1.767 |
| 512 | 5.255 | 4.853 |

### v2: 多 Warp 优化 (M=1)
- 尝试使用多个 warp 协作计算单个输出
- 意图提高 K 维度的并行度

**性能结果**:
| M | 延迟 (ms) | TFLOPS |
|---|-----------|--------|
| 1 | 0.025 | 1.992 |
| 8 | 0.226 | 1.764 |
| 512 | 5.242 | 4.864 |

**结论**: M=1 性能反而下降，因为 80 个 K 块对于 32 线程已经足够

### v3: 回归基础优化
- 回到 v1 的基本架构
- 优化线程块配置
- 细化 M 维度的边界条件

**性能结果**:
| M | 延迟 (ms) | TFLOPS |
|---|-----------|--------|
| 1 | 0.024 | 2.119 |
| 8 | 0.225 | 1.767 |
| 512 | 5.225 | 4.881 |

### v4: 最终优化版本 ⭐
- 精简代码结构
- 优化内存访问模式

## 4. 关键优化技术

### 4.1 M=1 内核
```cpp
// 激活值量化并缓存到共享内存
// 所有线程协作量化，然后每个 warp 独立计算一个 N 输出
- 1024 线程/块，32 warps
- 共享内存缓存激活值 (避免重复读取)
- DP4A 指令加速 INT8 点积
```

### 4.2 小批量内核 (M≤16)
```cpp
// 每个 warp 计算一个 (m, n) 输出元素
// 权重按需从全局内存加载
- 512 线程/块，16 warps
- Warp 内 K 维度归约
```

### 4.3 大批量内核
```cpp
// 权重分块缓存到共享内存
// 每个 warp 处理一行 M，lane 处理一列 N
- 256 线程/块
- TILE_M=8, TILE_N=32, K_BATCH=8
- 共享内存缓存权重块，减少全局内存访问
```

## 5. 性能瓶颈分析

### 内存带宽限制
- 权重数据量: 9728 × 80 × 34 = 26.4 MB
- RTX 4090 理论带宽: 1008 GB/s
- 实际带宽利用率: 40-50%

### M=8 性能较低的原因
- 权重被每个 M 行重复读取
- 共享内存容量不足以缓存所有权重
- 解决方案需要更复杂的跨行权重复用策略
